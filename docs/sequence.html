One decision question, end to end — federated retrieval, grounded synthesis, and a review-gated decision brief

"Should we start a new engagement before April?"

sequenceDiagram
  autonumber
  actor U as Consultant
  participant API as Beacon API
  participant R as Retriever
  participant S as Synthesizer
  participant D as Decision Layer

  U->>API: ask — "Should we start a new engagement before April?"
  API->>R: federated search across all 4 knowledge sources
  R-->>API: ranked, source-tagged documents + confidence
  API->>S: synthesize(question, documents, conversation context)
  Note over S: Claude when an API key is set ·
deterministic extractive fallback otherwise
  S-->>API: grounded answer with inline document-id citations
  API->>D: question detected as an operational decision
  Note over D: confidence is medium —
evidence partial, not conclusive
  D-->>API: decision brief + human-review flag raised
  API-->>U: recommendation + grounding + "human review recommended"

A plain question — same pipeline, no decision brief

sequenceDiagram
  autonumber
  actor U as Consultant
  participant API as Beacon API
  participant R as Retriever
  participant S as Synthesizer

  U->>API: ask — "What is the status of the Atterly Retail engagement?"
  API->>R: federated search (optionally scoped to one source)
  R-->>API: top documents across project records, client KB, deliverables
  API->>S: synthesize grounded answer
  S-->>API: answer + citations + confidence: high
  API-->>U: answer with every source document listed and traceable
  Note over U,API: not a decision question — returned as a direct answer
One pipeline, two shapes — plain questions get a direct grounded answer; decision questions get a structured brief Confidence drives the gate — thin or partial evidence raises a human-review flag rather than asserting false certainty Always traceable — the response always carries the exact documents it used, so any claim can be checked at source
