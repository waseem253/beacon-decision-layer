The architecture

flowchart TD
  USER([Consultant · a plain question or an operational decision]) --> API
  API{{"Beacon API · FastAPI

  SYN -.context.-> MEM[(Conversation memory

  SYN --> CLAUDE["Claude · when API key present"]
  SYN --> FALL["Deterministic extractive fallback · always on"]
  SYN --> DEC{Operational

  DEC -->|yes| BRIEF["Decision brief
recommendation + grounding + review gate"]
  DEC -->|no| ANS["Grounded answer"]
  BRIEF --> OUT[Answer + every source cited · full traceability]
  ANS --> OUT

Federated, not siloed — one query searches every connected source; each source is a swappable connector Grounded & traceable — every answer carries the document ids it rests on; nothing is asserted without a source Reliable by design — the extractive fallback means a provider outage degrades answer quality, never breaks the platform Decisions get a gate — when evidence is thin, the brief flags that a person should review before the business acts
